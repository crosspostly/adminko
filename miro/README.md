# Техническая документация Miro-интеграции

Этот раздел описывает, как работает связка Miro API и Telegram-бота.

## 🔑 Конфигурация
Все настройки доступа хранятся в `.env` (не путайте с `.env.example`):
*   `MIRO_ACCESS_TOKEN` — Ваш токен разработчика Miro.
*   `MIRO_BOARD_ID` — ID доски (последняя часть URL доски).

## ⚙️ Как работает маппинг
Для связи блоков Miro с логикой бота используется файл `memory/miro_mapping.json`.
1. Когда вы запускаете `export.py`, он создает блоки и записывает их `miro_id` в маппинг.
2. `manager.py` использует этот маппинг, чтобы понять, что блок с ID `345876...` — это, например, `Start_Msg`.
3. Даже если вы переименуете блок на доске, `miro_id` останется прежним, и связь не разорвется.

## 🛠 Устранение неполадок

### Бот не реагирует на новые стрелки
1. Убедитесь, что вы запустили `python3 miro/manager.py`.
2. Проверьте, что на стрелке есть текст. Если текста нет, бот посчитает это "авто-переходом" и сразу отправит следующее сообщение, не создавая кнопку.

### После экспорта всё перемешалось
Запустите `python3 miro/clean_all.py`, а затем снова `python3 miro/export.py`. Это полностью очистит доску и выстроит блоки по заданным в коде координатам.

### Ошибка Conflict в логах бота
Это значит, что запущен второй экземпляр бота. Остановите лишний процесс или службу:
```bash
sudo systemctl stop adminko_bot.service
```

## 📝 Разработчикам
Если вы добавляете новую специальную логику (например, оплату), добавьте проверку `if data == 'your_special_command'` в функцию `dynamic_callback_router` внутри `handlers.py`.
