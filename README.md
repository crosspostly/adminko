# Telegram-бот для Сервисного Центра "Админ-Ко"

Этот Telegram-бот предназначен для автоматизации взаимодействия с клиентами сервисного центра и управления системой лояльности.

## 🚀 Основной функционал

### 👤 Для клиентов
*   **Экспресс-тест (8 вопросов):** Интерактивная диагностика ПК с мгновенными советами.
*   **Система лояльности:** Автоматическое начисление **1000 приветственных бонусов** после теста (срок действия — 2 недели).
*   **Личный кабинет:** Просмотр баланса (бонусные и регулярные баллы) и генерация 6-значных кодов для оплаты.
*   **Каталог услуг:** Ремонт ПК/ноутбуков, IT-поддержка, видеонаблюдение.
*   **Заявка на диагностику:** Сбор контактных данных и уведомление администратора.

### 👨‍💼 Для менеджеров (Доступ по ID)
*   **Списание баллов (`/admin_redeem_points`):** Ввод кода клиента, суммы заказа и подтверждение списания у клиента в реальном времени.
*   **Верификация отзывов:** Менеджеры получают скриншоты отзывов и могут начислять по 500 баллов за каждый подтвержденный отзыв.

## 📊 Воронка лояльности и Retention (CRM-логика)

Бот реализует автоматическую воронку для вовлечения и удержания новых клиентов:

1.  **Deep Link (`start=maps_1500`):** Приветственный бонус **500 баллов** при переходе со специальных ссылок (например, с Яндекс Карт). Пользователь немедленно получает уведомление о начислении и кнопки для связи.
2.  **Отложенное предложение (2 часа):** Если пользователь еще не проходил тест, через 2 часа после первого контакта бот присылает сообщение с предложением пройти Экспресс-тест и получить **еще 1000 баллов**.
3.  **Сгорание предложения (12 часов):** Сообщение с предложением пройти тест автоматически удаляется через 12 часов после отправки. Сами баллы (если будут получены) живут 2 недели.
4.  **Напоминание о помощи (1 день):** Если пользователь пришел по deep-link, через 24 часа бот присылает ненавязчивое напоминание с предложением помощи специалиста.
5.  **Напоминание о сгорании (10 дней):** За 4 дня до конца срока действия (на 10-й день) бот напоминает пользователю, что его приветственные баллы скоро сгорят.
6.  **Запрос отзыва (24 часа после услуги):** После того как клиент воспользовался услугой и списал баллы, бот через сутки автоматически просит оставить отзыв на Яндекс/2GIS в обмен на **500 баллов**.

## 📂 Структура проекта
*   `bot_unified.py` — Точка входа, запускает и управляет ботом.
*   `handlers.py` — Основная логика обработки команд и сообщений от **пользователей и менеджеров**.
*   `core_logic.py` — Ядро бизнес-логики: начисление и расчет баллов, обработка тестов, планирование задач.
*   `timed_handlers.py` — Обработчики отложенных задач (отправка и удаление сообщений по таймеру).
*   `review_handlers.py` — Логика, связанная с системой отзывов.
*   `data_manager.py` — Слой для работы с базой данных SQLite.
*   `texts.py` — Хранилище всех текстовых сообщений для удобного редактирования.

## ⚙️ Установка и запуск

1. **Клонирование и окружение:**
   ```bash
   git clone https://github.com/crosspostly/adminko.git
   cd adminko
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   ```
2. **Настройка:**
   Укажите `TELEGRAM_BOT_TOKEN` в переменных окружения или `.env`. В `config.py` настройте `ADMIN_NOTIFICATION_CHAT_ID` и `MANAGER_IDS`.
3. **Запуск:**
   ```bash
   ./start_bot.sh
   ```
   Или вручную:
   ```bash
   python3 bot_unified.py
   ```

## 📝 Разработка и Miro
Инструменты в папке `miro/` позволяют синхронизировать сценарий бота с визуальной доской в Miro. Однако на данный момент бот работает на классических хендлерах для максимальной стабильности.
